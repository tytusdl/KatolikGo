Home 1-view — apa yang ada

  • 1 unified resource card — Level badge + XP bar + 5 Nyawa hearts + Token balance + Streak. Semua nampak sekali imbas.
  • Subject cards dipindah ke BAWAH resource card — XP/Nyawa dulu (resources yg perlu rancang), baru kategori kuiz. Tak overlap dengan hero — flat layout.
  • Refill countdown timer untuk Nyawa — "Auto-isi dalam 12 min 30 saat" dengan progress bar merah. Config dari LIVES_CONFIG.REFILL_MINUTES = 20 (full tick interval). Hidden bila lives = MAX.
  • Powerup counters (50/50, Hint, Skip) dibuang dari Home — hanya relevan dalam Quiz Play.
  • Recent quiz dikurangkan ke 2 items.

Refill timer config (dari src/constants/xp.constants.ts)

  • LIVES_CONFIG.REFILL_MINUTES = 20 — 1 nyawa setiap 20 minit (anchor dari livesLastLostAt)
  • LIVES_CONFIG.MAX = 5 — full bar bila cap
  • LIVES_CONFIG.REFILL_TOKEN_COST = 50 — instant refill via token
  • LIVES_CONFIG.AD_COOLDOWN_MIN = 5 — cooldown antara rewarded ad
  • Timer UI hanya muncul bila lives < MAX supaya user tahu bila free refill akan masuk

Quiz Play — design v10 (powerups jadi small utility pills)

  • Header: chevron-back + "Soalan X/Y" + small lives mini — back left, counter center, compact ❤ 5 right.
  • Powerups DIBUAT KECIL — bukan primary action surface, just utility buttons di tepi. Slim horizontal pill (icon 14px + name 10px) dengan brand color border (purple for 5050, gold for Hint). Tak stretch full width — left-aligned je.
  • Focus shifts to answers — powerups no longer compete untuk attention dengan answer buttons. Player focus kat question, powerups just ada kalau perlu.
  • Dual state shown side-by-side:
    • Correct state: A (Adam) green ✓, B+C faded (50/50 used), D plain. Timer at 00:12. Lives mini: ❤ 5.
    • Wrong state: A (Adam) green ✓, D (Abel) selected-wrong red ✗, B+C plain. Timer frozen at 00:07. Lives mini: ❤ 4 (coral heart).
  • 2 powerup: 5050 + Hint. Skip & Free Pass dibuang. Used state ditunjukkan dengan opacity 0.4 + ✓ corner badge.
  • Bottom sheet — explanation → small powerup row → Seterusnya.

Real code mapping (Level.tsx)

  • Line 35: QUESTION_TIME = 15 — seconds per question
  • Line 511-518: header with chevron-back calling router.back() directly (no Alert)
  • Line 544-548: LivesIndicator variant='banner' with pulseToken on wrong answer
  • Line 556-574: timer row with LinearGradient orange→red, MM:SS countdown
  • Line 577-584: hint card shown when hintRevealed && selectedAnswer === null
  • Line 600-622: 3 answer states — correct (green ✓), wrong selected (red ✗), other (faded), hidden (50/50)

Schema real dari all_questions.json

  • Setiap level ada array of QuizQuestion: {question, options[4 string], correctAnswer (index 0-3), explanation, level, difficulty, category, id, gameMode}
  • Tiada hint field dalam schema — Hint kena reveal correct answer atau generate clue, bukan tunjuk pre-existing hint text.

Powerup cost (dari code, enforced at spend time)

  • 50/50 — 2 token: FIFTY_FIFTY_COST = 2 (level.tsx L36)
  • Hint — 1 token: HINT_COST = 1 (level.tsx L37). Note: TOKEN_REWARDS.HINT_COST = 10 dalam xp.constants unused — drift.
  • Skip / Free Pass: tidak relevan — dibuang dari UI per user request.

Profile tab — design

  • Hero card pastel blue dengan avatar ring + display name + handle + level chip. Same visual language as Home.
  • 2×2 stats grid — Streak, Tahap Selesai, Rakan, Ketepatan. Compact icon+value+label.
  • Monthly progress card — 3 col stats (Kuiz/XP/Token) + linear bar (replaces the donut chart from original).
  • Menu items — Pencapaian / Parish Saya / Premium (gradient gold). Premium card emphasized dengan gold border.
  • Settings sub-menu — Notifikasi toggle, Bahasa, Privasi.
  • Log Keluar button outlined red.
  • Version text di paling bawah — hidden admin-unlock trigger location (long-press 5s untuk promote admin).

Profile vs reference

  • Reference image takde profile, jadi clone hanya style language (pastel blue hero, white cards, soft shadows).
  • Profile content driven by UserData fields: displayName, streakDays, levelsCompleted.length, friendsCount, accuracy, quizzesThisMonth, totalXP.
  • Real katolikgo Profile screen dah ada — mockup ni aligned dengan code tapi cleaner visually.

Next steps (implementation)

  • Skip: tambah handleSkip dalam quiz/[level].tsx, setSkipUsed setter, dan button handler macam Hint — spendToken(uid, 20, 'Skip powerup') + advance. Optional: skipsUsed counter untuk XP calculation (already in formula).
  • HINT_COST drift: align HINT_COST — sama ada kemas kini TOKEN_REWARDS.HINT_COST dari 10 → 1, atau import dari constants file dan padam hardcoded. Pick satu source of truth.
  • Hint reveal logic: tiada hint field dalam schema, so kena decide — highlight correct answer green ring? tunjuk clue text dari explanation truncated? atau generate dinamik?
  • Powerup counts: optional UserData fields untuk track owned quantities (separate dari tokens spend).

KatolikGo integration plan

  • tokenService.spendToken dah support token spend — cuma tambah action signature untuk spendTokensFor('skip' | 'hint' | '5050', userData).
  • livesService consumeLifeOnWrongAnswer boleh opt-out bila Free Pass aktif dalam session.
  • Powerup counts (50/50:3, Hint:5, Skip:2) boleh jadi field baru pada UserData — increment via purchase, decrement per usage.
